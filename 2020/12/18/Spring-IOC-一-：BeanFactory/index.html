Spring IOC (一) ： BeanFactory

1. 概述

Spring ioc容器有两条主要的设计路径。第一条设计路径从顶层的BeanFactory接口开始，经过HierarchicalBeanFactory，ConfigurableBeanFactory，AutowireCapableBeanFactory，AbstrcatBeanFactory等的继承与扩展后，实现了IOC容器基本功能。BeanFactory最常用的末端实现类为DefaultListableBeanFactory，其继承体系如下图所示。

2. Bean注册相关类

IOC容器管理的Bean可能会有别名，Spring有一套Bean注册管理的体系。接下来从源码入手分析这套体系。

2.1 AliasRegistry

别名注册的顶层入口为AliasRegistry。

2.2 BeanDefinitionRegistry

BeanDefinition的注册表接口

2.3 SimpleAliasRegistry

简单实现了AliasRegistry接口。主要用ConcurrentHashMap存储别名-真名键值对，通过对表的操作实现AliasRegistry的方法。

2.4 SingletonBeanRegistry

单例Bean的注册接口

2.5 DefaultSingletonBeanRegistry

继承自SimpleAliasRegistry同时实现了SingletonBeanRegistry接口，这个类非常重要，是后续解决循环依赖的关键。其在继承别名注册实现类的基础上，又实现了单例注册的功能，完成了将bean实例注册到单例注册表，移除，获取等相关功能。

2.6 FactoryBeanRegistrySupport

继承了DefaultSingletonBeanRegistry，父类DefaultSingletonBeanRegistry完成了所有普通Bean的注册功能，但是FactoryBean和普通Bean不一样，需要单独处理。
对需要区分FactoryBean和普通Bean的操作，先用instance of判断类型，再根据类型调用本类或父类的方法执行。

3. Bean工厂相关类

3.1 BeanFactory

Bean容器的顶层接口，规定了一些容器的基本方法。最主要的方法就是各种getBean方法。

3.2 HierarchicalBeanFactory

继承自BeanFactory，实现BeanFactory的层次结构，即BeanFactory的父子关系

3.3 ConfigurableBeanFactory

继承了HierarchicalBeanFactory和 SingletonBeanRegistry，兼具获取Bean、单例注册的功能。除此以外还增加了大量的功能，实现了存储类加载器、BeanPostProcessor、处理Bean依赖关系等功能。

3.4 AbstractBeanFactory

继承了FactoryBeanRegistrySupport：具有注册bean/FactoryBean的功能，比如，向bean容器中添加/获取bean等功能。
实现了ConfigurableBeanFactory：具有BeanFactory的功能，比如，类加载器、BeanPostProcessor、类型转换、属性编辑器、作用域、处理bean依赖关系等功能。
父类已经完成了Bean注册的相关功能，本类主要实现BeanFactory中的各种方法。

3.5 AutowireCapableBeanFactory

继承了BeanFactory接口，进行了一些功能。使BeanFactory中实例化的Bean具有自动装配功能。
自动装配功能包括：bean实例化，初始化，初始化前后调用BeanPostProcessors进行强化(ConfigurableBeanFactory是设置BeanPostProcessors而本接口是调用)。
一般不会主动调用该接口中的方法。

3.6 AbstractAutowireCapableBeanFactory

此类继承了AbstractBeanFactory并实现了AutowireCapableBeanFactory接口。

3.6.2 方法

该类的构造方法较为简单，可传入一个BeanFactory作为其parent。
本文主要解析一些较为重要的方法。

createBean(1)
实现了AutowireCapableBeanFactory接口的createBean方法，经典的创建和填充外部Bean对象的方法。

autowireBean
给Bean的属性赋值

createBean(2)

doCreateBean
创建Bean的真正逻辑，所作工作可分为以下几步：

  1. 调用本类的createBeanInstance方法返回BeanWrapper类型对象
  2. 调用本类的applyMergedBeanDefinitionPostProcessors
  3. 如果实例化bean的BeanDefinition是单例的，并且属性singletonsCurrentlyInCreation中包含当前正在创建的bean的beanName，那么会调用addSingletonFactory()将刚实例化好的bean直接存入三级缓存中(这个时候还没经历初始化)（解决循环依赖的核心部分，提前曝光单例bean，然后下一步如果发生了循环依赖，则可以解决）
  4. 调用本类的initializeBean进行初始化

createBeanInstance
根据实例化策略初步建立Bean实例

autowireConstructor
有参数的构造方法的实例化

instantiateBean
无参数的构造方法的实例化

getEarlyBeanReference
对半成品Bean通过后置处理器进行一些处理，之后再将这个Bean实例加入三级缓存

polulateBean
对Bean进行属性填充

autowireByName
按名称获取待注入的属性

autowireByType
按类型获取待注入的属性

applyPropertyValues
利用装配好的PropertyValues给Bean注入属性值

initializeBean
主要是调用用户设定的初始化方法，还有一些其他的初始化工作。

invokeAwareMethods
Spring中提供一些Aware相关接口，比如BeanFactoryAware、ApplicationContextAware等，实现这些Aware接口的bean被初始化后，可以取得一些相对应的资源。
如实现BeanFactoryAware的bean在初始化后，Spring容器将会注入BeanFactory的实例。

invokeInitMethods
客户定制的初始化方法除了使用配置init-method外，还可以使自定义的bean实现InitializingBean接口,并在afterPropertiesSet中实现自己的初始化业务逻辑。
init-method和afterPropertiesSet都是在初始化bean时执行，执行顺序是afterPropertiesSet先执行，init-method后执行。

4. ListableBeanFactory

继承了BeanFctory，扩展了BeanFactroy接口的功能，使其具有枚举所有Bean的功能，而不是根据name获取。
如果BeanFactory同时实现了HierarchicalBeanFactory，具有层次结构，但不会枚举其他BeanFactory中的Bean。

5. ConfigurableListableBeanFactory

继承了ListableBeanFactory, AutowireCapableBeanFactory, ConfigurableBeanFactory。
除了枚举所有bean以外，提供了用于分析和修改bean定义以及预先实例化单例的功能。
这个是IOC容器中功能最全面的接口，上面所有容器提到的所有功能，除了别名功能其他都有。

6. DefaultListableBeanFactory

实现了ConfigurableListableBeanFactory，BeanDefinitionRegistry，Serializable接口。IOC各种接口的终端，所有功能都有，比AbstractAutowireCapableBeanFactory以外，还多了Listable和注册BeanDefinition的功能。
